# Backend project setup - Cheatsheet

## Development environment

- **Version manager**: Use `nvm` with `.nvmrc` to lock Node version
- **TypeScript**: Enable strict mode from the start
- **Linter**: Catch errors early (ESLint, TSLint)
- **Formatter**: Prettier with shared config ensures consistent style
- **Path aliases**: Simplify imports (`@/utils` instead of `../../../utils`)
- **Debugger**: VS Code launch config or Docker debugging setup
- **Setup docs**: Clear local development instructions

## Common elements

- **Input validation**: Validate all user inputs (Zod, Joi, class-validator)
- **Logging**: Structured logs with levels. Consider JSON format for external log tools
- **Error handling**: Standardized error DTOs and error middleware
- **Authentication**: JWT, OAuth, or session-based auth
- **Authorization**: Role-based or permission-based access control (RBAC)
- **Database**: ORM/ODM with migration system and seed scripts
- **Storage**: Store and serve files or blobs (local, cloud providers like S3, GCS)
- **Testing**: Unit, integration, E2E tests with coverage thresholds
- **Feature flags**: Gradual rollouts (Split.io, LaunchDarkly)
- **Health checks**: `/health` endpoint for monitoring
- **Monitoring**: Metrics and alerting (Corologix, Datadog, New Relic, Prometheus)
- **Distributed tracing**: Request correlation across services
- **Messaging**: Event-driven architecture (RabbitMQ, Kafka)
- **Background workers**: Async job processing
- **Cache**: For example Redis for session storage and performance
- **Security**: SQL injection prevention, XSS protection, CSRF tokens
- **Documentation**: ADRs, OpenAPI specs, architecture diagrams (C4 model)

## Deployment

- **Containerization**: Dockerfile with multi-stage builds
- **CI/CD**: GitHub Actions with separate triggers for test types
- **Secrets**: Environment-specific secret management
- **Registry**: Container image storage (DockerHub, GCR, ECR)
- **Orchestration**: Kubernetes for complex deployments (optional)
- **Scaling**: Horizontal scaling with load balancer
- **Rate limiting**: Protect APIs from abuse
- **Encryption**: Enforce HTTPS, TLS for internal communication
- **Backups**: Automated database and data backups
